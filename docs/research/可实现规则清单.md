# 可实现规则清单

本文档用于后续 demo 开发时直接转成代码。

## 1. 可直接实现

### 表单字段

字段、取值、范围按 `官网规则反推.md` 第 2 节实现。

### 单位换算

```text
tc_mg_dl = tc_unit == 1 ? tc : tc / 0.0259
hdlc_mg_dl = hdlc_unit == 1 ? hdlc : hdlc / 0.0259
```

### 10 年风险

使用 `官网规则反推.md` 第 3 节的工程近似公式。

输出展示：

```text
display_percent = round(risk * 100, 1)
```

其中公式输出的 `risk` 是 `0-1` 小数。

### 理想风险

理想状态替换：

```text
tc = 200 mg/dl
hdlc = 40 mg/dl
sbp = 120
dbp = 80
drug = 0
waist = sex == 1 ? 90 : 85
dm = 0
csmoke = 0
fh_ascvd = 0
```

年龄、性别、南北方、城乡沿用原输入。

### 风险分层

```text
10 年风险:
  < 5%: 低危
  >= 5% 且 < 10%: 中危
  >= 10%: 高危

终生风险:
  < 32.8%: 低危
  >= 32.8%: 高危
```

### 温馨建议

按 `官网规则反推.md` 第 8 节实现。

## 2. 可临时实现但需标注

### 终生风险

当前没有公开原始完整公式。demo 可以临时使用：

```text
docs/research/lifetime_poly2_model.json
```

要求：

1. 仅用于 demo 近似。
2. UI 或代码注释中标注“工程拟合结果，非原始 China-PAR 终生风险公式”。
3. 如后续用于真实医学或核保决策，应替换为权威原始公式或服务。

## 3. 不建议实现为医学结论

以下内容目前没有足够证据直接作为权威医学规则：

1. 终生风险原始 Fine-Gray 模型系数。
2. 官网后端未公开的内部四舍五入和数值精度。
3. 官网未展示的异常输入处理，例如绕过前端提交非法性别。

```json
{
  "sex": 1,
  "age": 40,
  "region": 1,
  "area": 1,
  "waist": 80,
  "tc_unit": 2,
  "tc": 5.2,
  "hdlc_unit": 2,
  "hdlc": 1.3,
  "sbp": 145,
  "dbp": 80,
  "drug": 0,
  "dm": 0,
  "csmoke": 0,
  "fh_ascvd": 1
```
